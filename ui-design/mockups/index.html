FFL Playoffs

Fantasy Football League - Interactive UI Mockups

Traditional roster-based fantasy football with individual NFL player drafting

About These Mockups: These are responsive HTML mockups demonstrating the user interface for a traditional fantasy football application. Players draft individual NFL players across multiple positions (QB, RB, WR, TE, K, DEF, FLEX, Superflex) to build their rosters.
ONE-TIME DRAFT MODEL: Rosters are built ONCE before the season starts and are PERMANENTLY LOCKED when the first game begins. No waiver wire, no trades, no weekly lineup changes for the entire season.

PRE-LOCK STATE - Draft Phase

When: From league creation until the first NFL game of the season starts

User Actions Available:

  • Browse and search for NFL players
  • Draft players to fill all roster positions
  • View incomplete roster with "Add Player" options
  • Edit roster selections (drop/add players)
  • See countdown timer to permanent roster lock

Post-Lock State Mockups

After first game starts - rosters permanently locked

POST-LOCK STATE - Season Active (Read-Only)

When: From the moment the first NFL game starts until season ends

User Actions Available:

  • View roster (read-only) - NO modifications allowed
  • View player stats and weekly scores
  • View league standings and rankings
  • See "Roster Locked" badges and warnings
  • CANNOT: Add players, drop players, trade, use waiver wire, or make ANY roster changes

Roster Positions

The FFL Playoffs application supports the following roster positions:

QB - Quarterback RB - Running Back WR - Wide Receiver TE - Tight End K - Kicker DEF - Defense/Special Teams FLEX - RB/WR/TE SUPERFLEX - QB/RB/WR/TE
